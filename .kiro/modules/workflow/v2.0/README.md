## 🎯 功能说明

轻量化工作流系统，专为单人快速开发设计，支持：
- **原生功能开发流程**（4 阶段）：了解需求 → 分析需求 → 制定计划 → 落地执行
- **二次开发流程**（5 阶段）：评估代码 → 了解需求 → 分析需求 → 制定计划 → 落地执行
- **改一点测一点**：小步快跑，每个最小单元完成后立即测试
- **可选测试和文档**：根据需求灵活配置
- **保留核心资产**：状态管理（current_context.yaml）、目录规范、元信息管理

## ⚙️ 配置说明

### 全局开关

在 `.kiro/config.yaml` 中启用模块：

```yaml
modules:
  workflow:
    enabled: true
    version: v2.0
    priority: 200
```

### 默认行为配置

在 `config.yaml` 中配置：

```yaml
defaults:
  testing:
    enabled: false  # 默认关闭测试
    ask_user: true  # 每次询问用户
  
  documentation:
    enabled: true   # 默认生成文档
    ask_user: true  # 每次询问用户
```

## 📖 使用方法

### 触发工作流

当用户输入包含以下关键词时自动触发：
- **原生开发**：开发新功能、实现功能、创建模块、功能设计等
- **二次开发**：基于 XXX 代码、二次开发、参考、借鉴等

### 工作流选择

触发后会询问用户选择（中文）：
```
🤔 检测到工作流触发关键词

请选择工作流类型：
A. 进入原生功能开发流程（4 阶段，2 小时）
B. 进入二次开发流程（5 阶段，3 小时）
C. 不进入工作流，正常作答

请选择：A / B / C
```

### 需求复杂度分析

如果选择 A 或 B，系统会先分析需求复杂度（中文）：
- **简单需求**：直接进入工作流
- **中等需求**：建议生成需求探索文档
- **复杂需求**：建议拆分为多个子需求
- **过于复杂**：通知用户不适合此工作流

## 🔄 工作流程

### 完整流程图

```
用户输入触发关键词
    ↓
询问选择工作流类型（中文）
    ├─ A. 原生功能开发
    ├─ B. 二次开发
    └─ C. 不进入工作流
    ↓
需求复杂度分析（仅 A/B，中文）
    ├─ 简单：直接进入工作流
    ├─ 中等：建议生成探索文档
    ├─ 复杂：建议拆分需求
    └─ 过于复杂：通知用户
    ↓
进入对应工作流
```

### 原生功能开发流程（4 阶段）

1. **需求复杂度分析**（新增）
   - 分析需求规模、清晰度、独立性、技术复杂度
   - 根据复杂度决定是否生成探索文档
   - 如果需要拆分，引导用户拆分需求

2. **Phase 1：了解项目与需求**
   - 确认需求边界
   - 快速扫查可复用资产
   - 输出：1-3 行需求记录 + 复用资产清单

3. **Phase 2：分析需求**
   - 拆解为最小可执行单元
   - 主动澄清所有不确定点
   - 询问：冲突点、GUI 偏好、测试需求、文档需求

4. **Phase 3：制定执行计划**
   - 按"改一点测一点"拆分步骤
   - 对齐 kiro 规范
   - 输出：执行步骤 + 测试点

5. **Phase 4：落地执行**
   - 按计划执行最小单元开发
   - 每个单元完成后立即测试
   - 测试通过→下一个单元；不通过→回滚修复

6. **Phase 5：总结**（可选）
   - 根据 Phase 2 的选择决定是否生成文档

### 二次开发流程（5 阶段）

1. **需求复杂度分析**（新增）
   - 同原生开发流程

2. **Phase 0：评估相似代码**
   - 分析适配性（兼容性、改造成本）
   - 提取可复用片段
   - 确认改造风险
   - 输出：适配性结论 + 可复用片段 + 风险点

3. **Phase 1-4**：复用原生开发流程
   - Phase 1：将相似代码复用计划加入资产清单
   - Phase 3：将代码改造拆分为最小单元
   - Phase 4：新增兼容性测试

4. **Phase 5：总结**（可选）

## 💡 使用建议

### 何时使用原生开发流程？
- 从零开始开发新功能
- 功能相对独立
- 没有可参考的现成代码

### 何时使用二次开发流程？
- 有相似功能的代码可参考
- 需要改造现有代码
- 需要评估改造成本和风险

### 如何快速上手？
1. 在 Kiro 中输入触发关键词（如"开发新功能"）
2. 选择工作流类型（A/B/C）
3. 按照 AI 的引导逐步完成各阶段
4. 享受"改一点测一点"的快速迭代体验
