# Prompt 与 Python 代码联动验证指南

## 📋 文档信息

- **文档名称**: Prompt 与 Python 代码联动验证指南
- **创建日期**: 2026-03-03
- **版本**: v2.0（简化版）
- **状态**: ✅ 可用

## 🎯 目的

本文档说明如何验证 `.kiro/steering/main.md` 中的 Prompt 指令与 Python 动态加载器代码的一致性。

## 👥 角色分工

### 你的角色（用户）
- 阅读 main.md 中的 Prompt 指令
- 向 AI 提出测试问题
- 验证 AI 的回答是否符合 Prompt 描述

### AI 的角色（Kiro）
- 执行 Python 代码验证
- 回答你的测试问题
- 提供实际执行结果

## 🔍 验证方法

### 方法 1：Prompt 理解验证（你执行）

通过向 AI 提问，验证 AI 是否正确理解了 main.md 中的 Prompt 指令。

### 方法 2：Python 代码验证（AI 执行）

AI 运行 Python 代码，验证实际执行结果是否符合 Prompt 描述。

### 方法 3：一致性对比（共同完成）

对比 Prompt 描述和实际执行结果，确保一致性。

## 📊 验证清单

### 验证场景 1：Prompt 理解测试（你执行）

**目标**: 验证 AI 是否正确理解 main.md 中的动态加载器指令。

#### 测试问题清单

请向 AI（我）提出以下问题，验证我的回答是否符合 main.md 的描述：

**问题 1：加载流程**
```
提问：动态模块加载器的完整加载流程是什么？请列出所有步骤。

预期回答：应该列出 10 个步骤：
1. 读取顶层配置
2. 筛选启用的模块
3. 读取模块配置
4. 合并配置
5. 计算激活状态
6. 校验依赖关系
7. 按优先级排序
8. 加载 Steering 文件
9. 整合 Prompt 上下文
10. 输出加载日志
```

**问题 2：配置文件格式**
```
提问：顶层配置文件的格式是什么？需要包含哪些字段？

预期回答：应该描述 .kiro/config.yaml 的格式，包含：
- version 字段
- modules 字段
- 每个模块的 enabled、version、priority 字段
```

**问题 3：激活条件类型**
```
提问：模块支持哪些激活条件类型？请举例说明。

预期回答：应该列出 5 种类型：
1. always - 始终激活
2. directory_match - 目录匹配
3. file_type_match - 文件类型匹配
4. and - AND 逻辑组合
5. or - OR 逻辑组合
并提供示例
```

**问题 4：错误处理**
```
提问：如果配置文件不存在，系统应该如何处理？

预期回答：应该说明：
- 输出错误信息
- 使用空模块列表
- 不中断整个流程
```

**问题 5：优先级排序**
```
提问：模块按什么规则排序？如果优先级相同怎么办？

预期回答：应该说明：
- 按 priority 从高到低排序
- 优先级相同时按模块名称字母顺序排序
```

**问题 6：依赖关系**
```
提问：如果模块 A 依赖模块 B，但模块 B 被禁用了，会发生什么？

预期回答：应该说明：
- 输出警告信息
- 跳过模块 A
- 继续处理其他模块
```

**问题 7：循环依赖**
```
提问：如何处理循环依赖？

预期回答：应该说明：
- 检测循环依赖
- 输出错误信息
- 跳过所有涉及循环的模块
```

**问题 8：Steering 文件路径**
```
提问：workflow 模块的 Steering 文件路径是什么？为什么特殊？

预期回答：应该说明：
- 路径：.kiro/modules/workflow/v1.0/steering/workflow_selector.md
- 特殊原因：workflow 使用 workflow_selector.md 而不是 workflow.md
```

**问题 9：配置合并**
```
提问：顶层配置和模块配置冲突时，哪个优先？

预期回答：应该说明：
- 顶层配置优先级更高
- 模块配置作为补充
- 进行深度合并
```

**问题 10：日志格式**
```
提问：加载日志的格式是什么？

预期回答：应该说明：
- 格式：[时间] [级别] [组件] 消息内容
- 级别：INFO、WARNING、ERROR
- 组件：DynamicLoader
```

#### 如何验证

1. 向 AI 提出上述问题
2. 检查 AI 的回答是否符合预期
3. 如果回答不符合，说明 AI 没有正确理解 Prompt
4. 记录验证结果

## 💡 常见问题

### Q1：我需要手动运行 Python 代码吗？

**A**: 不需要！你只需要：
- 向 AI 提问（测试 Prompt 理解）
- 要求 AI 执行测试（AI 会运行 Python 代码）
- 对比结果与 main.md 的描述

### Q2：验证需要多长时间？

**A**: 大约 15-20 分钟：
- Prompt 理解测试：5-10 分钟
- Python 代码测试：5-10 分钟
- 生成报告：2-3 分钟

### Q3：如果发现不一致怎么办？

**A**: 记录问题，然后：
1. 确认是 Prompt 描述错误还是代码实现错误
2. 更新 main.md 或修复代码
3. 重新验证

### Q4：需要验证所有问题和测试吗？

**A**: 建议全部验证，但如果时间有限，可以：
- 至少验证 5 个 Prompt 理解问题
- 至少执行 4 个 Python 代码测试
- 重点验证核心功能

### Q5：验证频率是多少？

**A**: 建议：
- main.md 更新后：立即验证
- Python 代码重构后：立即验证
- 定期验证：每月一次
