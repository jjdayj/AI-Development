# Git 回滚功能 Phase 3 测试计划（步骤 7）

## 文档说明
本文档定义 Git 回滚功能 Phase 3 的完整测试计划，基于 Phase 2 MVP 测试经验和需求细化文档制定。

**重要**：本文档已根据需求文档（23-需求细化.md、22-架构决策.md）修正所有需求编号和配置项名称。

---

## 测试目标

### 主要目标
1. 验证 MVP 功能在实际使用中的表现
2. 验证 P1 需求的实现（扩展功能）
3. 确保边界情况得到正确处理
4. 验证用户体验是否符合预期
5. 确保功能关闭时不干扰正常对话

### 测试范围

#### 包含的功能
- ✅ P0 需求（MVP 核心功能）
- ✅ P1 需求（扩展功能，Phase 3 补充）
- ✅ 边界情况处理
- ✅ 错误处理
- ✅ 用户体验
- ✅ 回归测试（功能关闭时的行为）

#### 不包含的功能
- ❌ P2 需求（后续迭代）
- ❌ 性能测试（功能优先）
- ❌ 压力测试（不适用）

## 配置说明

### 备份分支配置（kiro.yaml）
根据需求文档 REQ-6.5.2，备份配置应该是：

```yaml
# Git 回滚配置
rollback:
  backup:
    enabled: true                    # 是否启用备份
    push_to_remote: false           # 是否推送到远程
    auto_delete_after_days: 7       # 自动删除天数（0=不删除）
```

**注意**：配置格式与 kiro.yaml 保持一致（YAML 格式）

## 风险与应对

### 风险 1：动态测试无法在当前对话执行
**影响**：延长测试周期
