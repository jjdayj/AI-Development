## 🚀 核心功能

### 功能 1：智能工作流识别 ⭐⭐⭐⭐⭐

**工作原理**:
```
用户提出需求
    ↓
检查手动覆盖？
    ├─ 是 → 使用用户指定的工作流
    └─ 否 → 分析文件路径
            ├─ 匹配成功 → 使用匹配的工作流
            └─ 匹配失败 → 分析关键词
                        ├─ 匹配成功 → 使用匹配的工作流
                        └─ 匹配失败 → 使用默认工作流
```

**匹配规则**:
- `.kiro/steering/*.md` → Prompt 工作流
- `.kiro/hooks/*.json` → Prompt 工作流
- `doc/project/*.md` → Prompt 工作流
- `**/*.py` → 代码工作流
- `scripts/*.py` → 代码工作流

### 功能 2：快速跳过机制 ⭐⭐⭐⭐⭐

**触发方式**:
1. 手动跳过关键词
   - "快速修改"、"简单修改"、"跳过流程"
   - "quick fix"、"simple change"

2. 自动跳过条件
   - 修改行数 < 10 行
   - 简单配置文件（*.yaml、*.json、*.txt）
   - 错别字、格式化、注释修改

**用户体验**:
```
用户："快速修改 kiro.yaml，添加一个配置项"

Kiro：
🤔 检测到快速修改关键词

这是一个简单修改吗？可以跳过完整的工作流吗？

请确认：
A. 是，跳过流程（5-10 分钟）
B. 否，执行完整流程（2.5-8 小时）

请选择：A / B
```

### 功能 3：强制总结反思（Phase 5）⭐⭐⭐⭐⭐

**5 个强制项**:
1. 问题识别与分类
2. 根本原因分析（5 Why）
3. 经验教训提取
4. 改进措施制定
5. 知识沉淀

**输出文档**:
1. **快速入门指南**（给新用户）
   - 阅读时间：5 分钟
   - 内容：核心特性、快速开始、常见问题

2. **详细说明文档**（给深度用户）
   - 阅读时间：15 分钟
   - 内容：完整流程、使用示例、配置详解

3. **总结与反思文档**（给团队）
   - 内容：问题分析、经验教训、改进措施
