# Q02: 顶层配置文件的格式是什么？

**提问时间**: 2026-03-03

## 问题

顶层配置文件的格式是什么？需要包含哪些字段？

## 回答

根据 `main.md` 文档，顶层配置文件（`.kiro/config.yaml`）的格式如下：

## 顶层配置文件格式

### 基本结构

```yaml
version: "2.0"

modules:
  {module-name}:
    enabled: true/false
    version: v{major}.{minor}[.{patch}][-variant]
    priority: {number}
```

### 必需字段

1. **version** (字符串)
   - 配置文件的版本号
   - 示例：`"2.0"`

2. **modules** (对象)
   - 包含所有模块的配置
   - 每个模块使用模块名作为键

### 每个模块的必需字段

1. **enabled** (布尔值)
   - 模块的全局开关
   - `true` - 启用模块（会进入激活条件判断）
   - `false` - 禁用模块（直接跳过）

2. **version** (字符串)
   - 模块的版本号
   - 格式：`v{major}.{minor}[.{patch}][-variant]`
   - 示例：`v1.0`, `v1.0.0`, `v1.0-simple`, `v2.1.3-alpha`

3. **priority** (数字)
   - 模块的优先级（数字越大优先级越高）
   - 建议范围：
     - 200-300：核心工作流模块
     - 100-199：功能模块
     - 50-99：辅助模块
     - 1-49：可选模块

## 完整示例

```yaml
version: "2.0"

modules:
  workflow:
    enabled: true
    version: v1.0
    priority: 200
  
  ai-dev:
    enabled: true
    version: v1.0
    priority: 100
  
  doc-save:
    enabled: false
    version: v1.0
    priority: 90
  
  git-commit:
    enabled: true
    version: v1.0
    priority: 80
  
  git-rollback:
    enabled: true
    version: v1.0
    priority: 70
```

## 使用场景

### 启用/禁用模块
```yaml
modules:
  workflow:
    enabled: true   # 启用
  doc-save:
    enabled: false  # 禁用
```

### 切换模块版本
```yaml
modules:
  workflow:
    enabled: true
    version: v1.0-simple  # 使用简化版
    priority: 200
  
  # 或者
  workflow:
    enabled: true
    version: v1.0-complex  # 使用复杂版
    priority: 200
```

### 调整优先级
```yaml
modules:
  workflow:
    enabled: true
    version: v1.0
    priority: 200  # 最高优先级，先加载
  
  ai-dev:
    enabled: true
    version: v1.0
    priority: 100  # 中等优先级
  
  git-commit:
    enabled: true
    version: v1.0
    priority: 80   # 较低优先级
```

## 注意事项

1. **YAML 格式**：必须是有效的 YAML 格式
2. **缩进**：使用 2 个空格缩进
3. **版本号**：遵循语义化版本规范
4. **优先级**：数字越大优先级越高，相同优先级按字母顺序
5. **模块名**：必须与 `.kiro/modules/` 下的目录名一致
