# Q03: 模块支持哪些激活条件类型？

**提问时间**: 2026-03-03

## 问题

模块支持哪些激活条件类型？请举例说明。

## 回答

根据 `main.md` 文档，模块支持以下 5 种激活条件类型：

## 模块激活条件类型

### 1. always - 始终激活

最简单的激活条件，模块始终处于激活状态。

```yaml
activation_conditions:
  always: true
```

**使用场景**：核心模块、基础功能模块

**示例**：
```yaml
# workflow 模块配置
activation_conditions:
  always: true
```

---

### 2. directory_match - 目录匹配

当当前工作目录匹配指定模式时激活。

```yaml
activation_conditions:
  directory_match:
    - "pattern1"
    - "pattern2"
```

**通配符支持**：
- `*` - 匹配单层目录或文件名
- `**` - 匹配多层目录

**示例 1：在 Steering 目录下激活**
```yaml
activation_conditions:
  directory_match:
    - ".kiro/steering/*"
    - ".kiro/modules/*/steering/*"
```

**示例 2：在文档目录下激活**
```yaml
activation_conditions:
  directory_match:
    - "doc/**/*"
    - "docs/**/*"
```

**匹配示例**：
- `.kiro/steering/*` 匹配 `.kiro/steering/main.md`
- `doc/**/*` 匹配 `doc/project/architecture/design.md`

---

### 3. file_type_match - 文件类型匹配

当当前编辑的文件类型匹配指定扩展名时激活。

```yaml
activation_conditions:
  file_type_match:
    - ".ext1"
    - ".ext2"
```

**示例 1：Markdown 文件激活**
```yaml
activation_conditions:
  file_type_match:
    - ".md"
```

**示例 2：配置文件激活**
```yaml
activation_conditions:
  file_type_match:
    - ".yaml"
    - ".json"
    - ".toml"
```

**示例 3：Python 文件激活**
```yaml
activation_conditions:
  file_type_match:
    - ".py"
```

---

### 4. and - AND 逻辑（所有条件都满足）

所有子条件都必须满足时才激活。

```yaml
activation_conditions:
  and:
    - condition1
    - condition2
```

**示例 1：在 Steering 目录下编辑 Markdown 文件**
```yaml
activation_conditions:
  and:
    - directory_match: [".kiro/steering/*"]
    - file_type_match: [".md"]
```

**示例 2：在文档目录下编辑配置文件**
```yaml
activation_conditions:
  and:
    - directory_match: ["doc/**/*"]
    - file_type_match: [".yaml", ".json"]
```

**匹配逻辑**：
- 当前目录：`.kiro/steering/`
- 当前文件：`main.md`
- 结果：✓ 激活（目录匹配 AND 文件类型匹配）

---

### 5. or - OR 逻辑（任一条件满足）

任一子条件满足时就激活。

```yaml
activation_conditions:
  or:
    - condition1
    - condition2
```

**示例 1：在 Steering 目录或编辑 Markdown 文件**
```yaml
activation_conditions:
  or:
    - directory_match: [".kiro/steering/*"]
    - file_type_match: [".md"]
```

**示例 2：在文档目录或编辑配置文件**
```yaml
activation_conditions:
  or:
    - directory_match: ["doc/**/*"]
    - file_type_match: [".yaml", ".json"]
```

**匹配逻辑**：
- 场景 A：当前目录 `.kiro/steering/`，文件 `main.py` → ✓ 激活（目录匹配）
- 场景 B：当前目录 `src/`，文件 `README.md` → ✓ 激活（文件类型匹配）
- 场景 C：当前目录 `src/`，文件 `main.py` → ✗ 不激活（都不匹配）

---

## 复杂嵌套示例

### 示例 1：复杂 OR 嵌套 AND

```yaml
activation_conditions:
  or:
    - and:
        - directory_match: [".kiro/steering/*"]
        - file_type_match: [".md"]
    - directory_match: ["doc/project/*"]
```

**匹配逻辑**：
- 场景 A：`.kiro/steering/main.md` → ✓ 激活（第一个 AND 条件满足）
- 场景 B：`doc/project/design.md` → ✓ 激活（第二个条件满足）
- 场景 C：`.kiro/steering/config.yaml` → ✗ 不激活（第一个 AND 不完全满足）

### 示例 2：多层嵌套

```yaml
activation_conditions:
  and:
    - or:
        - directory_match: [".kiro/steering/*"]
        - directory_match: [".kiro/modules/*/steering/*"]
    - file_type_match: [".md", ".yaml"]
```

**匹配逻辑**：
- 必须在 Steering 相关目录（OR 条件）
- 且必须是 Markdown 或 YAML 文件

## 激活状态计算公式

```
最终激活状态 = 全局开关 (enabled) AND 激活条件匹配
```

**示例**：
```yaml
# .kiro/config.yaml
modules:
  doc-save:
    enabled: true  # 全局开关：启用
    version: v1.0
    priority: 90

# .kiro/modules/doc-save/v1.0/config.yaml
activation_conditions:
  directory_match: ["doc/**/*"]  # 激活条件：在 doc 目录下
```

**计算结果**：
- 在 `doc/project/` 目录：✓ 激活（enabled=true AND 目录匹配）
- 在 `src/` 目录：✗ 不激活（enabled=true BUT 目录不匹配）
- 如果 `enabled: false`：✗ 不激活（无论目录是否匹配）

---

## 最佳实践

1. **核心模块**：使用 `always: true`
2. **特定目录功能**：使用 `directory_match`
3. **特定文件类型功能**：使用 `file_type_match`
4. **精确控制**：使用 `and` 组合多个条件
5. **灵活激活**：使用 `or` 提供多种激活路径
6. **避免过度复杂**：嵌套层级不要超过 3 层
