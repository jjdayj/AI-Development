# Q04: 如果配置文件不存在，系统应该如何处理？

**提问时间**: 2026-03-03

## 问题

如果配置文件不存在，系统应该如何处理？

## 回答

根据 `main.md` 文档，系统对配置文件不存在的情况有明确的错误处理策略：

## 配置文件不存在的处理策略

### 1. 顶层配置文件不存在（.kiro/config.yaml）

**步骤 1：读取顶层配置**

```
错误处理：
- 如果配置文件不存在，输出错误信息并使用空模块列表
- 如果 YAML 格式错误，输出错误详情并使用空模块列表
```

**处理流程**：
```
1. 尝试读取 .kiro/config.yaml
2. 如果文件不存在：
   - 输出错误日志：[ERROR] 顶层配置文件不存在: .kiro/config.yaml
   - 使用空模块列表：modules = {}
   - 继续执行（不会崩溃）
3. 结果：没有模块被加载
```

**日志示例**：
```
[ERROR] [DynamicLoader] ============================================================
[ERROR] [DynamicLoader] 步骤 1: 读取顶层配置
[ERROR] [DynamicLoader] ============================================================
[ERROR] [DynamicLoader] 读取顶层配置文件: .kiro/config.yaml
[ERROR] [DynamicLoader] ✗ 配置文件不存在
[INFO]  [DynamicLoader] 使用空模块列表继续执行

[INFO]  [DynamicLoader] ============================================================
[INFO]  [DynamicLoader] 模块加载完成: 0 个成功, 0 个失败
[INFO]  [DynamicLoader] ============================================================
```

---

### 2. 模块配置文件不存在（.kiro/modules/{module}/{version}/config.yaml）

**步骤 3：读取模块配置**

```
错误处理：
- 如果模块配置不存在，使用默认配置（always: true）
- 如果 YAML 格式错误，使用默认配置
```

**默认配置**：
```yaml
activation_conditions:
  always: true

dependencies: []

module_specific_config: {}
```

**处理流程**：
```
1. 尝试读取 .kiro/modules/workflow/v1.0/config.yaml
2. 如果文件不存在：
   - 输出警告日志：[WARNING] 模块配置不存在，使用默认配置
   - 使用默认配置：{activation_conditions: {always: true}}
   - 继续处理该模块
3. 结果：模块使用默认配置继续加载
```

**日志示例**：
```
[INFO]  [DynamicLoader] 步骤 3: 读取模块配置
[INFO]  [DynamicLoader] ------------------------------------------------------------
[INFO]  [DynamicLoader] 读取模块配置: workflow (v1.0)
[WARNING] [DynamicLoader] ⚠ 模块配置文件不存在: .kiro/modules/workflow/v1.0/config.yaml
[INFO]  [DynamicLoader] 使用默认配置: {activation_conditions: {always: true}}
```

---

### 3. Steering 文件不存在（.kiro/modules/{module}/{version}/steering/{module}.md）

**步骤 8：加载 Steering 文件**

```
错误处理：
- 如果 Steering 文件不存在，输出警告并跳过该模块
- 如果文件读取失败，输出错误并继续处理下一个模块
```

**处理流程**：
```
1. 尝试读取 .kiro/modules/workflow/v1.0/steering/workflow_selector.md
2. 如果文件不存在：
   - 输出警告日志：[WARNING] Steering 文件不存在，跳过该模块
   - 跳过该模块，不加载其 Steering 内容
   - 继续处理下一个模块
3. 结果：该模块的 Steering 规则不会被加载
```

**日志示例**：
```

[INFO]  [DynamicLoader] ------------------------------------------------------------
[INFO]  [DynamicLoader] 加载 Steering: workflow (v1.0)
[WARNING] [DynamicLoader] ⚠ Steering 文件不存在: .kiro/modules/workflow/v1.0/steering/workflow_selector.md
[INFO]  [DynamicLoader] 跳过该模块

[INFO]  [DynamicLoader] 加载 Steering: ai-dev (v1.0)
[INFO]  [DynamicLoader] ✓ Steering 加载成功
```

## 完整错误处理矩阵

| 配置文件类型 | 不存在时的处理 | 格式错误时的处理 | 是否继续执行 | 影响范围 |
|------------|--------------|----------------|------------|---------|
| **顶层配置** (.kiro/config.yaml) | 使用空模块列表 | 使用空模块列表 | ✓ 继续 | 没有模块被加载 |
| **模块配置** (.kiro/modules/{module}/{version}/config.yaml) | 使用默认配置 | 使用默认配置 | ✓ 继续 | 该模块使用默认激活条件 |
| **Steering 文件** (.kiro/modules/{module}/{version}/steering/{module}.md) | 跳过该模块 | 跳过该模块 | ✓ 继续 | 该模块的规则不生效 |

## 最佳实践

### 1. 配置文件完整性检查

建议在项目初始化时检查必需的配置文件：

```python
required_files = [
    '.kiro/config.yaml',
    '.kiro/modules/workflow/v1.0/config.yaml',
    '.kiro/modules/workflow/v1.0/steering/workflow_selector.md',
]

for file_path in required_files:
    if not os.path.exists(file_path):
        print(f"[WARNING] 缺失配置文件: {file_path}")
```

### 2. 使用模板初始化

使用 `.kiro/templates/` 中的模板创建新模块：

```bash
# 复制模板
cp -r .kiro/templates/module/ .kiro/modules/new-module/v1.0/

# 编辑配置
vim .kiro/modules/new-module/v1.0/config.yaml
vim .kiro/modules/new-module/v1.0/steering/new-module.md
```

### 3. 优雅降级

系统设计遵循优雅降级原则：
- 配置缺失不会导致系统崩溃
- 使用合理的默认值
- 输出清晰的警告信息
- 继续处理其他模块

这确保了系统的健壮性和可用性。
