# Q05: 模块按什么规则排序？如果优先级相同怎么办？

**提问时间**: 2026-03-03

## 问题

模块按什么规则排序？如果优先级相同怎么办？

## 回答

根据 `main.md` 文档，模块排序规则非常明确：

## 模块排序规则（步骤 7）

### 主要排序规则

**步骤 7：按优先级排序**

```
将通过依赖校验的模块按以下规则排序：
1. 按 priority 从高到低排序
2. 如果 priority 相同，按模块名称字母顺序排序
```

---

## 详细排序逻辑

### 规则 1：按优先级从高到低

数字越大，优先级越高，排序越靠前。

**示例**：
```yaml
modules:
  workflow:
    priority: 200
  ai-dev:
    priority: 100
  git-commit:
    priority: 80
```

**排序结果**：
```
1. workflow (200)
2. ai-dev (100)
3. git-commit (80)
```

---

### 规则 2：优先级相同时按字母顺序

当多个模块的 `priority` 值相同时，按模块名称的字母顺序排序。

**示例 1：三个模块优先级相同**
```yaml
modules:
  git-commit:
    priority: 100
  doc-save:
    priority: 100
  ai-dev:
    priority: 100
```

**排序结果**（按字母顺序）：
```
1. ai-dev (100)      # 'a' 开头
2. doc-save (100)    # 'd' 开头
3. git-commit (100)  # 'g' 开头
```

**示例 2：混合优先级**
```yaml
modules:
  workflow:
    priority: 200
  git-commit:
    priority: 100
  doc-save:
    priority: 100
  ai-dev:
    priority: 100
  git-rollback:
    priority: 70
```

**排序结果**：
```
1. workflow (200)        # 最高优先级
2. ai-dev (100)          # 优先级 100，字母 'a'
3. doc-save (100)        # 优先级 100，字母 'd'
4. git-commit (100)      # 优先级 100，字母 'g'
5. git-rollback (70)     # 最低优先级
```

---

## 排序的重要性

### 1. 加载顺序

模块按排序后的顺序加载 Steering 文件。

```
加载顺序 = 排序顺序
```

**示例**：
```
排序结果：workflow → ai-dev → git-commit

加载顺序：
1. 加载 .kiro/modules/workflow/v1.0/steering/workflow_selector.md
2. 加载 .kiro/modules/ai-dev/v1.0/steering/ai_dev.md
3. 加载 .kiro/modules/git-commit/v1.0/steering/git_commit.md
```

---

### 2. 规则优先级

优先级高的模块规则优先生效（在规则冲突时）。

**冲突处理规则**（来自 `main.md`）：
```markdown
## 冲突处理规则

- 优先级高的模块规则优先
- 优先级相同时，字母顺序靠前的模块规则优先
```

**示例场景**：
```yaml
# workflow 模块规则（优先级 200）
工作流选择：自动检测

# ai-dev 模块规则（优先级 100）
工作流选择：始终使用 Prompt 工作流
```

**冲突解决**：
- workflow 的规则优先（优先级 200 > 100）
- 最终行为：自动检测工作流

## 排序算法伪代码

```python
def sort_modules(modules):
    """
    对模块进行排序
    
    规则：
    1. 按 priority 从高到低排序
    2. 如果 priority 相同，按模块名称字母顺序排序
    """
    return sorted(
        modules,
        key=lambda m: (-m['priority'], m['name'])
    )
```

**说明**：
- `-m['priority']`：负号表示从高到低排序
- `m['name']`：字母顺序（默认从小到大）

**示例**：
```python
modules = [
    {'name': 'git-commit', 'priority': 80},
    {'name': 'workflow', 'priority': 200},
    {'name': 'ai-dev', 'priority': 100},
    {'name': 'doc-save', 'priority': 100},
]

sorted_modules = sort_modules(modules)

# 结果：
# [
#     {'name': 'workflow', 'priority': 200},
#     {'name': 'ai-dev', 'priority': 100},
#     {'name': 'doc-save', 'priority': 100},
#     {'name': 'git-commit', 'priority': 80},
# ]
```

## 建议的优先级分配

根据 `main.md` 的最佳实践：

```yaml
# 核心工作流模块（200-300）
workflow:
  priority: 200

# 功能模块（100-199）
ai-dev:
  priority: 100

doc-save:
  priority: 90

git-commit:
  priority: 80

# 辅助模块（50-99）
git-rollback:
  priority: 70

# 可选模块（1-49）
experimental-feature:
  priority: 10
```

**优先级范围建议**：
- **200-300**：核心工作流模块
- **100-199**：功能模块
- **50-99**：辅助模块
- **1-49**：可选模块

---

## 总结

**排序规则**：
1. 主要规则：按 `priority` 从高到低
2. 次要规则：优先级相同时按模块名称字母顺序

**影响**：
- 决定模块加载顺序
- 决定规则冲突时的优先级
- 决定 Prompt 整合顺序

**最佳实践**：
- 为不同类型的模块分配不同的优先级范围
- 避免所有模块使用相同的优先级
- 核心模块使用更高的优先级
