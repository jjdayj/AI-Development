# Q07: 如何处理循环依赖？

**提问时间**: 2026-03-03

## 问题

如何处理循环依赖？

## 回答

根据 `main.md` 文档，系统对循环依赖有明确的检测和处理机制。

## 循环依赖处理机制（步骤 6）

### 步骤 6：校验依赖关系

```
检查每个激活模块的依赖：
1. 确保所有依赖的模块都在激活列表中
2. 检测循环依赖（A 依赖 B，B 依赖 A）
3. 依赖不满足的模块不应该被加载

错误处理：
- 缺失依赖：输出警告，跳过该模块
- 循环依赖：输出错误，跳过所有涉及的模块
```

---

## 循环依赖的定义

### 什么是循环依赖？

循环依赖是指模块之间形成依赖环路，导致无法确定加载顺序。

**类型 1：直接循环依赖（2 个模块）**
```
module-a → module-b
module-b → module-a
```

**类型 2：间接循环依赖（3 个模块）**
```
module-a → module-b
module-b → module-c
module-c → module-a
```

**类型 3：复杂循环依赖（多个模块）**
```
module-a → module-b
module-b → module-c
module-c → module-d
module-d → module-b  (形成环路)
```

## 处理流程

### 场景 1：简单循环依赖（A ↔ B）

**配置**：
```yaml
# .kiro/config.yaml
modules:
  module-a:
    enabled: true
    version: v1.0
    priority: 100
  
  module-b:
    enabled: true
    version: v1.0
    priority: 90

# .kiro/modules/module-a/v1.0/config.yaml
dependencies:
  - module-b

# .kiro/modules/module-b/v1.0/config.yaml
dependencies:
  - module-a
```

**依赖关系图**：
```
module-a → module-b
    ↑         ↓
    └─────────┘
```

**处理流程**：
```
步骤 2：筛选启用的模块
- 启用列表：[module-a, module-b]

步骤 6：校验依赖关系
1. 开始检测循环依赖
2. 从 module-a 开始 DFS：
   - 访问 module-a
   - 检查依赖 module-b
   - 访问 module-b
   - 检查依赖 module-a
   - module-a 已在递归栈中 ❌
   - 检测到循环：{module-a, module-b}

3. 输出错误日志
4. 跳过所有涉及的模块
```

**日志输出**：
```

[INFO]  [DynamicLoader] 步骤 6: 校验依赖关系
[INFO]  [DynamicLoader] ============================================================
[ERROR] [DynamicLoader] ✗ 检测到循环依赖: {module-a, module-b}
[WARNING] [DynamicLoader] ⚠ 模块 module-a 存在循环依赖，跳过加载
[WARNING] [DynamicLoader] ⚠ 模块 module-b 存在循环依赖，跳过加载

[INFO]  [DynamicLoader] ============================================================
[INFO]  [DynamicLoader] 模块加载完成: 0 个成功, 2 个失败
[INFO]  [DynamicLoader] ============================================================
[INFO]  [DynamicLoader] 失败模块:
[INFO]  [DynamicLoader]   ✗ module-a: 循环依赖 {module-a, module-b}
[INFO]  [DynamicLoader]   ✗ module-b: 循环依赖 {module-a, module-b}
```

**结果**：
- 检测到循环依赖
- module-a 和 module-b 都被跳过
- 0 个模块成功加载

## 解决循环依赖的方法

### 方法 1：移除依赖关系

**问题**：
```yaml
# module-a 依赖 module-b
dependencies:
  - module-b

# module-b 依赖 module-a
dependencies:
  - module-a
```

**解决方案**：移除其中一个依赖
```yaml
# module-a 依赖 module-b
dependencies:
  - module-b

# module-b 不依赖 module-a
dependencies: []
```

**结果**：
```
module-a → module-b
```
正常的单向依赖，可以成功加载。

---

### 方法 2：引入中间模块

**问题**：
```
module-a ↔ module-b
```

**解决方案**：创建共享的基础模块
```yaml
# 创建 module-base
# .kiro/modules/module-base/v1.0/config.yaml
dependencies: []

# module-a 依赖 module-base
dependencies:
  - module-base

# module-b 依赖 module-base
dependencies:
  - module-base
```

**结果**：
```
module-a → module-base
module-b → module-base
```
两个模块都依赖基础模块，没有循环依赖。

---

### 方法 3：重构模块职责

**问题**：模块 A 和 B 相互依赖，说明职责划分不清晰。

**解决方案**：
1. 分析两个模块的功能
2. 提取共同功能到新模块
3. 重新划分职责边界

**示例**：
```
原始设计：
- module-auth: 用户认证（依赖 module-user）
- module-user: 用户管理（依赖 module-auth）

重构后：
- module-auth-core: 核心认证逻辑（无依赖）
- module-auth: 认证功能（依赖 module-auth-core）
- module-user: 用户管理（依赖 module-auth-core）
```

## 循环依赖检测的完整实现

### Python 实现示例

```python
class CircularDependencyDetector:
    """循环依赖检测器"""
    
    def __init__(self, modules):
        self.modules = modules
        self.visited = set()
        self.rec_stack = set()
        self.circular_groups = []
    
    def detect(self):
        """检测所有循环依赖"""
        for module_name in self.modules:
            if module_name not in self.visited:
                path = []
                self._dfs(module_name, path)
        
        return self.circular_groups
    
    def _dfs(self, module_name, path):
        """深度优先搜索"""
        # 检查是否在递归栈中（循环依赖）
        if module_name in self.rec_stack:
            # 找到循环的起点
            cycle_start = path.index(module_name)
            cycle = set(path[cycle_start:])
            
            # 避免重复记录相同的循环
            if cycle not in self.circular_groups:
                self.circular_groups.append(cycle)
            return True
        
        # 已访问过，跳过
        if module_name in self.visited:
            return False
        
        # 标记为已访问和在递归栈中
        self.visited.add(module_name)
        self.rec_stack.add(module_name)
        path.append(module_name)
        
        # 检查所有依赖
        module = self.modules.get(module_name, {})
        dependencies = module.get('dependencies', [])
        
        for dep in dependencies:
            if dep in self.modules:
                if self._dfs(dep, path):
                    return True
        
        # 从递归栈中移除
        self.rec_stack.remove(module_name)
        path.pop()
        return False

# 使用示例
modules = {
    'module-a': {'dependencies': ['module-b']},
    'module-b': {'dependencies': ['module-a']},
    'module-c': {'dependencies': []},
}

detector = CircularDependencyDetector(modules)
circular_groups = detector.detect()

if circular_groups:
    for group in circular_groups:
        print(f"[ERROR] 检测到循环依赖: {group}")
else:
    print("[INFO] 未检测到循环依赖")
```

## 最佳实践

### 1. 避免循环依赖的设计原则

**原则 1：单向依赖**
```
✓ 好的设计：
module-a → module-b → module-c

✗ 坏的设计：
module-a ↔ module-b
```

**原则 2：分层架构**
```
Layer 3: 应用层模块（依赖 Layer 2）
Layer 2: 功能层模块（依赖 Layer 1）
Layer 1: 基础层模块（无依赖）
```

**原则 3：依赖倒置**
```
高层模块不应该依赖低层模块，两者都应该依赖抽象。
```

---

### 2. 定期检查依赖关系

在 CI/CD 流程中添加依赖检查：

```bash
# 检查循环依赖
python scripts/check_dependencies.py

# 如果发现循环依赖，构建失败
if [ $? -ne 0 ]; then
    echo "发现循环依赖，构建失败"
    exit 1
fi
```

## 总结

**循环依赖处理机制**：

1. **检测**：使用深度优先搜索（DFS）检测循环依赖
2. **报告**：输出错误日志，明确指出循环依赖的模块组
3. **处理**：跳过所有涉及循环依赖的模块
4. **继续**：系统不会崩溃，继续处理其他模块

**关键点**：
- 循环依赖会导致所有涉及的模块都被跳过
- 系统会输出清晰的错误信息
- 不会影响其他正常模块的加载
- 需要手动修复循环依赖（移除依赖或重构）

**最佳实践**：
- 设计时避免循环依赖
- 使用分层架构
- 定期检查依赖关系
- 文档化依赖原因
