# Q10: 加载日志的格式是什么？

**提问时间**: 2026-03-03

## 问题

加载日志的格式是什么？

## 回答

根据 `main.md` 文档，加载日志有明确的格式规范。

## 加载日志格式（步骤 10）

### 步骤 10：输出加载日志

```
输出清晰的加载日志，包含：
- 加载开始标题
- 每个模块的详细信息（名称、版本、优先级、激活状态）
- 加载完成摘要（总数、成功数、失败数）
```

---

## 日志格式规范

### 1. 日志级别

```
[INFO]    - 正常信息
[WARNING] - 警告信息（模块跳过、配置缺失等）
[ERROR]   - 错误信息（配置错误、依赖问题等）
```

### 2. 日志前缀

```
[级别] [DynamicLoader] 消息内容
```

**示例**：
```
[INFO]  [DynamicLoader] 步骤 1: 读取顶层配置
[WARNING] [DynamicLoader] ⚠ 模块配置不存在，使用默认配置
[ERROR] [DynamicLoader] ✗ 检测到循环依赖: {module-a, module-b}
```

### 3. 分隔线

```
============================================================  (60 个等号)
------------------------------------------------------------  (60 个短横线)
```

**用途**：
- `====` 用于主要步骤的开始
- `----` 用于子步骤的分隔

## 日志符号约定

### 状态符号

```
✓  - 成功 (U+2713)
✗  - 失败 (U+2717)
⚠  - 警告 (U+26A0)
→  - 箭头 (U+2192)
```

### 使用示例

```
[INFO] [DynamicLoader]   ✓ 配置读取成功
[ERROR] [DynamicLoader]   ✗ 配置文件不存在
[WARNING] [DynamicLoader]   ⚠ 模块配置不存在，使用默认配置
[INFO] [DynamicLoader]   module-a → module-b
```

## Python 日志实现示例

### Logger 类

```python
class DynamicLoaderLogger:
    """动态加载器日志工具"""
    
    def __init__(self):
        self.prefix = "[DynamicLoader]"
    
    def info(self, message, indent=0):
        """输出信息日志"""
        spaces = "  " * indent
        print(f"[INFO]  {self.prefix} {spaces}{message}")
    
    def warning(self, message, indent=0):
        """输出警告日志"""
        spaces = "  " * indent
        print(f"[WARNING] {self.prefix} {spaces}{message}")
    
    def error(self, message, indent=0):
        """输出错误日志"""
        spaces = "  " * indent
        print(f"[ERROR] {self.prefix} {spaces}{message}")
    
    def separator(self, char="="):
        """输出分隔线"""
        line = char * 60
        self.info(line)
    
    def step_header(self, step_num, step_name):
        """输出步骤标题"""
        self.separator("=")
        self.info(f"步骤 {step_num}: {step_name}")
        self.separator("=")
    
    def sub_separator(self):
        """输出子步骤分隔线"""
        self.separator("-")
    
    def success(self, message, indent=0):
        """输出成功信息"""
        self.info(f"✓ {message}", indent)
    
    def failure(self, message, indent=0):
        """输出失败信息"""
        self.error(f"✗ {message}", indent)
    
    def warn(self, message, indent=0):
        """输出警告信息"""
        self.warning(f"⚠ {message}", indent)

# 使用示例
logger = DynamicLoaderLogger()

# 步骤标题
logger.step_header(1, "读取顶层配置")

# 正常信息
logger.info("读取顶层配置文件: .kiro/config.yaml")

# 成功信息
logger.success("配置读取成功，共 4 个模块")

# 子步骤分隔
logger.sub_separator()

# 缩进信息
logger.info("筛选结果: 3 个模块启用")
logger.success("workflow (v1.0) - 优先级: 200", indent=1)
logger.success("ai-dev (v1.0) - 优先级: 100", indent=1)

# 警告信息
logger.warn("模块配置不存在，使用默认配置", indent=1)

# 错误信息
logger.failure("检测到循环依赖: {module-a, module-b}")
```

## 日志最佳实践

### 1. 清晰的步骤划分

每个步骤都有明确的开始和结束标记。

```
[INFO] [DynamicLoader] ============================================================
[INFO] [DynamicLoader] 步骤 1: 读取顶层配置
[INFO] [DynamicLoader] ============================================================
... 步骤内容 ...
```

### 2. 一致的缩进

使用 2 个空格表示层级关系。

```
[INFO] [DynamicLoader] 读取模块配置: workflow (v1.0)
[INFO] [DynamicLoader]   路径: .kiro/modules/workflow/v1.0/config.yaml
[INFO] [DynamicLoader]   ✓ 配置读取成功
```

### 3. 明确的状态标识

使用符号清晰表示成功、失败、警告。

```
✓ - 成功
✗ - 失败
⚠ - 警告
```

### 4. 详细的错误信息

错误日志包含足够的上下文信息。

```
[ERROR] [DynamicLoader] ✗ 检测到循环依赖: {module-a, module-b}
[WARNING] [DynamicLoader] ⚠ 模块 module-a 存在循环依赖，跳过加载
[WARNING] [DynamicLoader] ⚠ 模块 module-b 存在循环依赖，跳过加载
```

### 5. 完整的摘要信息

最后输出完整的加载摘要。

```
[INFO] [DynamicLoader] ============================================================
[INFO] [DynamicLoader] 模块加载完成: 2 个成功, 1 个失败
[INFO] [DynamicLoader] ============================================================
[INFO] [DynamicLoader] 成功加载的模块:
[INFO] [DynamicLoader]   ✓ workflow (v1.0) - 优先级: 200
[INFO] [DynamicLoader]   ✓ git-commit (v1.0) - 优先级: 80
[INFO] [DynamicLoader] 
[INFO] [DynamicLoader] 失败的模块:
[INFO] [DynamicLoader]   ✗ ai-dev (v1.0) - 缺少依赖 [doc-save]
```

## 总结

**日志格式规范**：
- 级别：`[INFO]`, `[WARNING]`, `[ERROR]`
- 前缀：`[DynamicLoader]`
- 分隔线：`====` (主步骤), `----` (子步骤)
- 符号：`✓` (成功), `✗` (失败), `⚠` (警告)
- 缩进：2 个空格表示层级

**日志结构**：
1. 步骤标题（带分隔线）
2. 详细信息（带缩进）
3. 状态标识（带符号）
4. 最终摘要（成功/失败统计）

**最佳实践**：
- 清晰的步骤划分
- 一致的缩进规则
- 明确的状态标识
- 详细的错误信息
- 完整的摘要信息
