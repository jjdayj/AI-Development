## 实施步骤

### 步骤 1：配置文件修改（5分钟）
**目标**：在 kiro.yaml 中添加 git_auto_rollback 开关

**操作**：
1. 打开 `.kiro/kiro.yaml`
2. 在 `experimental` 部分添加：
   ```yaml
   git_auto_rollback: false  # 默认关闭
   ```
3. 保存文件

**验证**：
- [ ] 配置文件格式正确
- [ ] 可以被 YAML 解析器读取

---

### 步骤 2：创建 Steering 规则（70分钟）
**目标**：创建 Git 回滚的 Steering 规则文件

**文件位置**：`.kiro/modules/ai-dev/v1.0/steering/git_rollback.md`

**内容结构**：
```markdown
---
inclusion: always
description: Git 回滚功能规则
---

# Git 回滚功能规则

## 1. 配置检查
检查 kiro.yaml 中的 git_auto_rollback 配置

## 2. 触发关键词识别
识别：回滚、撤销、恢复、rollback、revert

## 3. 回滚方式选择
默认使用 git revert

## 4. 回滚信息展示
展示提交列表、影响文件、风险提示

## 5. 用户确认
等待用户输入 yes/no

## 6. 执行回滚
执行 git revert HEAD
推送到远程
```

**实施细节**：
- 使用自然语言描述流程
- 提供清晰的命令示例
- 包含错误处理提示

## MVP 简化说明

### 简化 1：只支持回滚最近一次提交
- 不解析"回滚3次"
- 不支持指定 commit ID
- 固定使用 `git revert HEAD`

### 简化 2：不创建备份分支
- 依赖 git revert 的可恢复性
- Phase 3 再添加备份功能

### 简化 3：边界情况简化处理
- 工作目录有修改：直接提示错误
- 回滚冲突：提示手动解决
- 网络失败：提示手动推送

### 简化 4：提示信息简化
- 功能关闭：简单提示
- 回滚信息：只显示最近一次提交
- 风险提示：固定3条
