# Prompt 基座项目 - 架构说明

## 📋 项目简介

这是一个模块化的 Prompt 基座项目，用于管理和组织各种 AI 辅助开发功能模块。

**当前版本**: v2.0  
**架构重构日期**: 2026-03-03  
**状态**: ✅ 生产可用

---

## 🎯 核心理念

1. **模块化**: 每个功能独立模块，支持多版本共存
2. **需求驱动**: 需求与实现分离，可追溯
3. **配置灵活**: 顶层控开关，模块控细节
4. **文档规范**: 统一的文档结构和模板
5. **版本管理**: 语义化版本，支持迭代升级

## 🔧 已安装模块

### 1. workflow (v1.0) - 双轨工作流
- **优先级**: 200（最高）
- **状态**: ✅ 启用
- **功能**: 智能识别并执行 Prompt 工作流或代码工作流
- **特性**:
  - 自动识别工作流类型
  - 快速跳过机制
  - 强制总结反思（Phase 5）
  - 生成三份文档

### 2. ai-dev (v1.0) - AI 开发自动化
- **优先级**: 100
- **状态**: ✅ 启用
- **功能**: AI 驱动的开发自动化工具
- **特性**:
  - 自动生成代码
  - 自动运行测试
  - 代码格式化

### 3. git-commit (v1.0) - Git 提交管理
- **优先级**: 80
- **状态**: ✅ 启用
- **功能**: Git 提交主题化管理
- **特性**:
  - 自动读取需求 meta.yaml
  - 生成规范提交信息
  - 提交前安全检查
  - 自动推送

### 4. git-rollback (v1.0) - Git 回滚功能
- **优先级**: 70
- **状态**: ✅ 启用
- **功能**: 通过自然语言指令进行 Git 回滚
- **特性**:
  - 回滚单次/多次提交
  - 自动创建备份分支
  - 安全确认机制

### 5. doc-save (v1.0) - 会话文档自动保存
- **优先级**: 90
- **状态**: ❌ 禁用（默认）
- **功能**: 自动保存每次对话的操作文档和对话记录
- **特性**:
  - 保存对话记录
  - 保存操作文档
  - 生成会话总览

## ⚙️ 配置管理

### 顶层配置：`.kiro/config.yaml`

```yaml
# 模块全局开关
modules:
  workflow:
    enabled: true
    version: v1.0
    priority: 200
  
  ai-dev:
    enabled: true
    version: v1.0
    priority: 100
  
  git-commit:
    enabled: true
    version: v1.0
    priority: 80
  
  git-rollback:
    enabled: true
    version: v1.0
    priority: 70
  
  doc-save:
    enabled: false  # 默认禁用
    version: v1.0
    priority: 90
```

### 模块配置

每个模块都有独立的配置文件：
- 位置: `.kiro/modules/{module}/{version}/config.yaml`
- 作用: 管理模块内部细节配置
- 示例: `.kiro/modules/git-rollback/v1.0/config.yaml`

## 📖 使用指南

### 启用/禁用模块

编辑 `.kiro/config.yaml`：

```yaml
modules:
  {module-name}:
    enabled: true  # 改为 false 禁用
```

### 调整模块优先级

修改 `priority` 值（数字越大优先级越高）：

```yaml
modules:
  my-module:
    priority: 150  # 调整优先级
```

### 切换模块版本

修改 `version` 值：

```yaml
modules:
  my-module:
    version: v2.0  # 切换到 v2.0
```

### 查看模块文档

每个模块都有完整的文档：
- 模块信息: `.kiro/modules/{module}/{version}/README.md`
- 元信息: `.kiro/modules/{module}/{version}/meta.yaml`
- 配置: `.kiro/modules/{module}/{version}/config.yaml`
- Steering: `.kiro/modules/{module}/{version}/steering/`

## 🚀 创建新模块

### 步骤 1：复制模板

```powershell
# 复制模块模板
Copy-Item -Recurse ".kiro/templates/module" ".kiro/modules/my-module/v1.0"

# 复制需求模板
Copy-Item -Recurse ".kiro/templates/requirement" "requirements/my-module/v1.0"
```

### 步骤 2：编辑元信息

编辑 `.kiro/modules/my-module/v1.0/meta.yaml`：

```yaml
name: my-module
display_name: 我的模块
version: v1.0
author: yangzhuo
date: 2026-03-03
description: 模块功能描述
category: general
dependencies: []
```

### 步骤 3：配置模块

编辑 `.kiro/modules/my-module/v1.0/config.yaml`：

```yaml
# 模块特定配置
feature_enabled: true
option_1: value1
```

### 步骤 4：创建 Steering

在 `.kiro/modules/my-module/v1.0/steering/` 目录下创建 Steering 文件。

### 步骤 5：注册模块

编辑 `.kiro/config.yaml`，添加模块：

```yaml
modules:
  my-module:
    enabled: true
    version: v1.0
    priority: 50
```
