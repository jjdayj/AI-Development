## 🎯 核心功能

### 功能 1：智能工作流识别

**识别方式**:
1. 手动覆盖（最高优先级）
   - 用户明确指定工作流类型
2. 文件路径匹配
   - `.kiro/steering/*.md` → Prompt 工作流
   - `**/*.py` → 代码工作流
3. 关键词匹配
   - "Steering 规则" → Prompt 工作流
   - "Python 代码" → 代码工作流
4. 默认工作流
   - 使用 `workflow.default` 配置

### 功能 2：快速跳过机制

**触发条件**:
- 手动跳过关键词："快速修改"、"简单修改"、"跳过流程"
- 自动跳过条件：
  - 修改行数 < 10 行
  - 简单配置文件（*.yaml、*.json、*.txt）
  - 错别字、格式化、注释修改

**用户确认**:
- 检测到跳过条件后，询问用户确认
- 用户选择 A（跳过）或 B（执行完整流程）

### 功能 3：强制总结反思（Phase 5）

**5 个强制项**:
1. 问题识别与分类
2. 根本原因分析（5 Why）
3. 经验教训提取
4. 改进措施制定
5. 知识沉淀

**输出文档**:
1. 快速入门指南（给新用户）
2. 详细说明文档（给深度用户）
3. 总结与反思文档（给团队）

## 📊 配置说明

### kiro.yaml 配置

```yaml
workflow:
  # 默认工作流
  default: "prompt"
  
  # 自动切换规则
  auto_switch:
    # Prompt 工作流
    - pattern: ".kiro/steering/*.md"
      workflow: "prompt"
    - pattern: ".kiro/hooks/*.json"
      workflow: "prompt"
    - pattern: "doc/project/*.md"
      workflow: "prompt"
    
    # 代码工作流
    - pattern: "**/*.py"
      workflow: "code"
    - pattern: "scripts/*.py"
      workflow: "code"
  
  # 快速跳过配置
  quick_skip:
    enabled: true
    
    # 手动跳过关键词
    manual_skip_keywords:
      - "快速修改"
      - "简单修改"
      - "跳过流程"
      - "quick fix"
      - "simple change"
    
    # 自动跳过条件
    auto_skip_conditions:
      - type: "line_count"
        threshold: 10
      - type: "file_type"
        patterns: ["*.yaml", "*.json", "*.txt"]
      - type: "change_type"
        types: ["typo", "format", "comment"]
```
