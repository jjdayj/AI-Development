# VibCoding 流程优化方案

**方案创建时间**: 2026-03-02  
**方案版本**: v1.0  
**目标**: 优化 VibCoding 开发流程，防止类似 Git 提交工作流的错误再次发生

---

## 📋 优化目标

### 核心诉求 1：补充强制检查项
在现有 VibCoding 流程基础上，补充以下强制检查项：
- ✅ 目标用户识别（Agent / 人类用户 / 两者）
- ✅ 使用场景识别（Agent 内部调用 / 独立工具 / 两者）
- ✅ 触发方式识别（Steering 规则 / 命令行 / API / 其他）
- ✅ 交互方式识别（Agent 对话 / 命令行交互 / GUI / 其他）
- ✅ 集成方式识别（如果目标用户包含 Agent）

### 核心诉求 2：主动提问机制
要求 Agent 在遇到不确定的信息时：
- ✅ 在文档中记录问题（现有行为）
- ✅ 主动在对话框向用户提问（新增行为）
- ✅ 等待用户明确回答后再继续

---

## 🎯 实施方案概览

### 方案 A：创建新的 VibCoding Steering 规则（推荐）

**优点**：
- ✅ Agent 自动加载，100% 生效
- ✅ 强制执行检查流程
- ✅ 与现有文档独立，不影响历史记录

**缺点**：
- ⚠️ 需要创建新文件
- ⚠️ 需要测试验证

**工作量评估**：2-3 小时

### 方案 B：更新现有 VibCoding 文档 + 创建 Steering 规则

**优点**：
- ✅ 文档和规则同步更新
- ✅ 保持一致性

**缺点**：
- ⚠️ 工作量较大
- ⚠️ 需要同时维护两份内容

**工作量评估**：4-5 小时

---

## 📝 推荐方案详细说明（方案 A）

### 第 1 步：创建 VibCoding Steering 规则

**文件位置**：`.kiro/steering/vibcoding_development_process.md`

**文件结构**：

```markdown
---
inclusion: always
description: VibCoding 开发流程规则（强制检查点 + 主动提问版）
---

# VibCoding 开发流程规则

## 🎯 规则说明

本规则定义了 VibCoding（振动式编码）开发流程的强制执行规范。

**核心特性**：
- ✅ 强制检查目标用户和使用场景
- ✅ 遇到不确定信息时主动提问
- ✅ 防止需求理解错误

---

## 🔔 触发条件

当用户提出以下类型的需求时，自动触发 VibCoding 流程：

### 关键词识别
- "开发新功能"、"实现功能"、"添加功能"
- "创建模块"、"新建模块"
- "设计功能"、"功能设计"
- 或任何涉及新功能开发的需求

### 排除场景
- 简单的配置修改（如修改 YAML 配置）
- 纯文档编写（如更新 README）
- Bug 修复（除非涉及架构调整）

---

## 📋 Phase 1: 探索与决策（强制执行）

### 步骤 1.1：需求探索 - 目标用户识别（强制）

**执行时机**：在开始任何开发工作之前

**强制问题清单**：

#### 问题 1：目标用户是谁？（必答）
```
🤔 请明确：这个功能的目标用户是谁？

选项：
A. Agent（AI 助手内部调用）
B. 人类用户（手动运行脚本/命令）
C. 两者都是

请选择：A / B / C
```

**如果用户回答不明确**：
```
⚠️ 目标用户不明确，无法继续

请明确说明：
- 如果选 A（Agent）：功能将通过 Steering 规则或 Hook 自动触发
- 如果选 B（人类用户）：功能将作为独立工具，用户手动运行
- 如果选 C（两者）：需要设计两种使用方式

请重新回答：A / B / C
```

#### 问题 2：使用场景是什么？（必答）
```
🤔 请明确：这个功能的使用场景是什么？

选项：
A. Agent 内部调用（通过 Steering 规则触发）
B. 独立工具（用户手动运行）
C. 两者都支持

请选择：A / B / C
```

#### 问题 3：触发方式是什么？（必答）
```
🤔 请明确：这个功能如何触发？

选项（可多选）：
A. Steering 规则识别关键词
B. Hook 规则（特定事件触发）
C. 命令行参数
D. API 调用
E. GUI 界面
F. 其他（请说明）

请选择：A / B / C / D / E / F
```

#### 问题 4：交互方式是什么？（必答）
```
🤔 请明确：这个功能如何与用户交互？

选项：
A. Agent 与用户对话
B. 命令行交互
C. GUI 界面
D. 无交互（自动执行）
E. 其他（请说明）

请选择：A / B / C / D / E
```

#### 问题 5：集成方式是什么？（如果目标用户包含 Agent）
```
🤔 请明确：这个功能如何与 Agent 集成？

选项：
A. Steering 规则（Agent 自动识别关键词）
B. Hook 规则（Agent 在特定事件时触发）
C. Python 模块（Agent 调用 Python 脚本）
D. 直接执行命令（Agent 执行 git/shell 命令）
E. 其他（请说明）

请选择：A / B / C / D / E
```

**重要提示**：
```
⚠️ 关键区分：

如果目标用户是 Agent：
- ✅ 正确理解：Agent 在对话中识别关键词，自动执行操作
- ❌ 错误理解：用户手动运行 Python 脚本

如果使用场景是 Agent 内部调用：
- ✅ 正确实现：Steering 规则 + Agent 执行命令
- ❌ 错误实现：独立的命令行工具

参考示例：
- Git 回滚功能：目标用户=Agent，使用场景=Agent 内部调用
- AI Development：目标用户=人类用户，使用场景=独立工具
```
```

**预计工作量**：30 分钟（编写规则）

**实现方式**：
```json
{
  "name": "VibCoding 流程自动触发",
  "version": "1.0.0",
  "when": {
    "type": "promptSubmit",
    "patterns": ["开发.*功能", "实现.*功能", "创建.*模块"]

  "then": {
    "type": "askAgent",

}
```
