# 工作流文档模板说明

本文档定义了 Phase 5 结束后需要生成的三份文档的完整模板。

**重要说明**：
- 这些文档是针对当前开发的功能模块的说明文档
- 不是针对整个 Prompt 基座项目的文档
- 而是针对本次需求完成的具体功能的使用指南

**示例**：
- 如果开发的是"Git 回滚功能"，生成的是"Git 回滚功能 - 快速入门指南"
- 如果开发的是"双轨工作流"，生成的是"双轨工作流 - 快速入门指南"
- 每个功能都有自己独立的使用指南和详细说明

---

## 📋 文档生成规则

每个项目完成后，必须生成三份文档：

1. **快速入门指南**（给新用户看的）⭐⭐⭐
2. **详细说明文档**（给深度用户看的）⭐⭐⭐  
3. **总结与反思文档**（给团队看的）⭐⭐

---

## 📄 文档 1：快速入门指南

**文件名**：`doc/project/YYYYMMDD_项目名-快速入门指南.md`

**目标读者**：第一次使用的新用户

**阅读时间**：5 分钟

**参考模板**：`doc/project/20260302_双轨工作流-快速入门指南.md`

**生成要点**：
- 用"问题-解决方案"的方式说明价值
- 用"旧版 vs 新版"对比（如果适用）
- 用实际效果和优势来说明特性
- 提供 3 步快速开始指南
- 列出 5-8 个常见问题

**完整模板**：见 `doc/project/20260302_双轨工作流-快速入门指南.md`

**关键章节**：
- 👋 欢迎使用（1-2 句话说明）
- 🎯 这个系统能帮你做什么？（2-3 个核心问题）
- 🚀 核心特性（3-5 个，用星级标注）
- 📖 快速开始（3 步）
- 🎯 常见问题（5-8 个）
- 📚 进阶阅读
- 🎉 开始使用

## 📄 文档 2：详细说明文档

**文件名**：`doc/project/YYYYMMDD_项目名-详细说明.md`

**目标读者**：想深入了解的用户

**阅读时间**：15 分钟

**参考模板**：`doc/project/20260302_双轨工作流-核心内容说明.md`

**生成要点**：
- 详细说明每个特性的背景、问题、解决方案
- 提供完整的工作流程说明

- 提供完整的配置说明
- 提供高级用法和故障排查

**完整模板**：见 `doc/project/20260302_双轨工作流-核心内容说明.md`

**关键章节**：
- 🎯 这是什么？（详细说明 + 核心理念）
- ✨ 核心特性（所有特性，包含技术细节）
- 🚀 完整工作流程（流程图 + 每个阶段详解）
- 📖 使用示例（3-5 个完整示例）
- ⚙️ 配置说明（完整配置 + 每项详解）
- 🎯 常见问题（10-15 个）
- 🔧 高级用法
- 📚 相关文档
- 🎯 最佳实践
- 🐛 故障排查

## 📄 文档 3：总结与反思文档

**文件名**：`doc/project/YYYYMMDD_项目名-总结与反思.md`

**目标读者**：团队成员

**用途**：团队复盘、知识积累

**内容**：
- 项目背景和结果
- 问题识别与分类

- 经验教训
- 改进措施
- 知识沉淀
- 成本统计
- 未来建议

## 🔧 生成逻辑

### 1. 自动提取信息

从各个 Phase 提取关键信息：

**Phase 1（探索与决策）**：
- 目标用户
- 使用场景
- 触发方式
- 交互方式
- 集成方式

**Phase 2（实现）**：
- 核心功能
- 关键特性
- 实现方式
- 技术选型

**Phase 3（验证/完善）**：
- 测试结果
- 边界情况处理
- 性能指标

**Phase 4（审查与发布）**：
- 审查结果
- 发布信息
- 版本号

**Phase 5（总结与反思）**：
- 遇到的问题
- 根本原因
- 经验教训
- 改进措施

### 2. 人性化改写

**快速入门指南**：
- 用"问题-解决方案"方式
- 用"旧版 vs 新版"对比
- 用实际效果说明价值
- 用简单的语言，避免术语

**详细说明文档**：
- 保留技术细节
- 提供完整的配置说明
- 提供多个使用示例
- 提供故障排查指南

**总结与反思**：
- 客观记录问题
- 深入分析原因
- 提取可行动的教训
- 制定具体的改进措施

### 3. 结构化呈现

**快速入门指南结构**：
```
1. 欢迎使用（1-2 句话）
2. 能帮你做什么？（2-3 个问题）
3. 核心特性（3-5 个）
4. 快速开始（3 步）
5. 常见问题（5-8 个）
6. 进阶阅读
7. 开始使用
```

**详细说明文档结构**：
```
1. 这是什么？（详细说明）
2. 核心特性（所有特性）
3. 完整工作流程
4. 使用示例（3-5 个）
5. 配置说明（完整）
6. 常见问题（10-15 个）
7. 高级用法
8. 相关文档
9. 最佳实践
10. 故障排查
```

**总结与反思文档结构**：
```
1. 项目背景
2. 项目结果
3. 问题识别
4. 根本原因分析
5. 经验教训
6. 改进措施
7. 知识沉淀
8. 成本统计
9. 未来建议
```

### 4. 质量检查

**快速入门指南**：
- ✅ 阅读时间 ≤ 5 分钟
- ✅ 语言简单易懂
- ✅ 示例可以直接运行
- ✅ 3 步即可开始使用

**详细说明文档**：
- ✅ 阅读时间 ≤ 15 分钟
- ✅ 所有配置都有说明
- ✅ 所有示例都完整
- ✅ 所有问题都有答案

**总结与反思**：
- ✅ 问题分析客观
- ✅ 原因分析深入
- ✅ 教训可行动
- ✅ 改进措施具体
