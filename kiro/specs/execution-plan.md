## 执行注意事项

### 1. 测试策略差异
- **动态模块加载器**：完整测试，确保基座稳定性
- **Git 提交自动化**：简单测试，快速验证核心功能
- **VibCoding 工作流**：手动验证，灵活调整 Prompt 逻辑

### 2. 格式校验机制
- 动态模块加载器完成后，立即使用 `output_validator.py` 验证输出格式
- Git 提交模块开始前，运行格式校验确保兼容性
- VibCoding 工作流开始前，再次运行格式校验

### 3. Checkpoint 必须通过
- 每个 Checkpoint 都是质量关卡
- 如果 Checkpoint 未通过，不要继续下一阶段
- 及时向用户报告问题

### 4. 文档同步更新
- 代码实现的同时更新文档
- 数据格式变更必须同步到规范文档
- 保持文档与代码一致

### 5. 沟通机制
- 遇到问题及时向用户提问
- 每个里程碑完成后向用户报告
- 定期同步进度和风险

## 风险管理

### 高风险项
1. **数据格式不兼容**：通过输出格式校验工具降低风险
2. **测试覆盖不足**：动态模块加载器必须完整测试
3. **Prompt 逻辑不可执行**：通过 Prompt 可执行性测试降低风险

### 中风险项
1. **任务顺序错误**：严格按照执行清单顺序执行
2. **依赖关系未校验**：每个模块开始前运行前置校验

### 低风险项
1. **文档不完整**：每个阶段同步更新文档
2. **配置冲突**：通过配置合并规则处理
