## 常见问题

### Q1：用户名可以修改吗？
A：可以，但不建议频繁修改。修改路径：Settings → Account → Change username

### Q2：邮箱可以修改吗？
A：可以。可以添加多个邮箱，并设置主邮箱。路径：Settings → Emails

### Q3：忘记密码怎么办？
A：在登录页面点击"Forgot password"，通过邮箱重置密码

### Q4：免费版够用吗？
A：对于个人学习和开发，免费版完全够用

### Q5：可以创建多少个仓库？
A：无限制，公开和私有仓库都可以创建无限个

## 注意事项

1. **用户名选择**
   - 用户名会显示在所有提交记录中
   - 建议选择专业、易记的用户名

   - 使用强密码

   - 建议使用密码管理器

3. **邮箱验证**
   - 必须验证邮箱才能使用完整功能
   - 如果没收到验证邮件，检查垃圾邮件箱

4. **两步验证**
   - 强烈建议开启
   - 可以防止账号被盗

## GitHub 基本概念

### Repository（仓库）
- 存放项目代码的地方
- 可以是公开的（Public）或私有的（Private）

### Commit（提交）
- 代码的一次保存记录
- 包含修改内容和提交信息

### Branch（分支）
- 代码的不同版本
- 默认分支通常是 `main` 或 `master`

### Pull Request（PR）
- 请求将代码合并到主分支
- 用于代码审查和协作

### Issue（议题）
- 用于跟踪任务、功能请求或 bug
- 团队协作的重要工具
